# Cover Layout Recipes

## Editorial Magazine x E-ink Recipes (M01-M16)

These structures work for any topic that wants a magazine-feature pace. The mode is a visual stance, not a content filter.

**Content density rule**: on 1080×1440 cards, content must cover ≥75% of canvas height. Pure-whitespace bands >15% canvas height need a stated reason.

### M01 Cover: Magazine Issue Cover
Best for: Rednote page 1, article-card covers.
- Top issue row: category, date, account label
- Large serif title, 2-4 lines
- One large image occupying 35%-55% of page
- Bottom issue strip with 3-5 points

### M02 Field Note Photo
Best for: outdoor, object, hardware, real-world observation.
- Large documentary photo
- Narrow caption column or bottom caption band
- One short takeaway in large type

### M03 Editorial Essay Split
Best for: explaining one idea with nuance.
- Left: large title or pull quote
- Right: 2-3 short paragraphs or numbered fragments
- Thin rule between columns

### M04 Pull Quote / Thesis
Best for: core sentence or conclusion.
- Large quote across the page
- Small source/context row
- Optional tiny note or issue marker

### M05 Checklist / Buying Guide
Best for: practical content.
- Header title
- 4-6 rows, each with number, item, consequence
- Optional small photo crop

### M06 Evidence Wall
Best for: multiple screenshots, references, small images.
- 2×2 or 3-column image grid
- Each image has a short caption
- One larger headline anchors interpretation

### M07 Closing Note
Best for: final page.
- Big takeaway title (≤2 lines)
- 4-6 ledger items with sub-lines
- Closing block: pull-quote OR signature OR CTA

### M08 Tall Ledger
Best for: lists, roles, pros/cons, gear items.
- Header title
- 4-6 full-width rows (118-170px each)
- Left index/marginalia column, right title + consequence

### M09 Atmospheric Thesis
Best for: sparse but important points.
- WebGL/ink-flow background visible
- One very large thesis or quote
- 1-2 supporting notes
- Small issue metadata

### M10 Evidence Feature
Best for: supplied screenshots/photos.
- Large image occupying 45%-65% of vertical canvas
- Headline and lead above or beside
- Bottom caption band with 2-3 takeaways

### M11 Marginalia Essay
Best for: nuanced explanation with moderate text.
- Wide editorial title
- Main column with 2-3 paragraphs
- Narrow marginal column with keywords/quotes
- Hairline vertical rule between columns

### M12 Section Divider
Best for: mid-carousel breath between dense pages.
- WebGL ink-flow background
- One mono kicker like "Act II"
- One large serif h-display naming section
- One short subtitle in serif italic

### M13 Hero Question
Best for: last page or sharp pivot.
- One question in very large type
- Optional context line below
- Rest is intentional whitespace

### M14 Profile Card
Best for: person introductions, team members.
- Photo well (3:4 or square)
- Name in h-display
- Title/role in mono kicker
- 1-2 lines of bio in lead

### M15 Comparison Table
Best for: before/after, feature comparison.
- Two or three columns
- Each column has a header + 4-6 rows
- Accent highlights the "winner" or recommended option

### M16 Timeline
Best for: chronological stories, process evolution.
- Vertical timeline with 4-6 nodes
- Each node: date + event + brief description
- Accent color for timeline line and nodes

## Swiss International Recipes (S01-S12)

Swiss recipes use strict grid, hairline rules, one accent color. "The larger, the lighter" — ≥200px → weight 200.

### S01 KPI Tower
Best for: product metrics, dashboard highlights.
- 3-4 large KPI numbers (h-hero or h-statement)
- Each with mono label above
- Optional small comparison or delta below

### S02 H-Bar Chart
Best for: comparisons, rankings, scores.
- 4-6 horizontal bars with labels
- One accent-colored bar for highlight
- Mono values at bar end

### S03 Matrix + Hero
Best for: feature grid with hero statement.
- Large hero statement top (h-xl)
- 2×3 or 3×3 card-fill matrix below
- Each card: icon + title + brief text

### S04 Process Flow
Best for: step-by-step processes.
- 3-5 steps connected by lines/arrows
- Each step: number + title + brief description
- Accent color on current/important step

### S05 Comparison Slider
Best for: before/after, with/without.
- Two panels side by side
- Clear visual division (hairline or gap)
- Labels in mono, descriptions in body

### S06 Icon Grid
Best for: feature lists, capability showcases.
- 2×3 or 3×3 grid of icon + label pairs
- Lucide icons
- Optional brief description under label

### S07 Stats Grid
Best for: data overview.
- 2×2 grid of large stats
- Each stat: h-xl number + mono label
- Accent background on one highlighted stat

### S08 Timeline Swiss
Best for: release notes, version history.
- Vertical timeline with 4-6 entries
- Each entry: date (mono) + title (h-md) + brief note
- Accent line on left edge

### S09 Feature List
Best for: product features, tool capabilities.
- 4-6 features, each with icon + title + description
- Card-fill containers for each feature
- Accent border on highlighted feature

### S10 Pricing Table
Best for: pricing, plan comparison.
- 2-3 plan columns
- Each: plan name + price + feature list
- Accent background on recommended plan

### S11 Team Grid
Best for: team introductions.
- 2×3 or 3×3 grid of profile cards
- Each card: photo + name + role
- Mono labels, clean spacing

### S12 CTA Banner
Best for: call-to-action, sign-up, download.
- Large accent-colored banner
- h-xl statement
- One action button or URL
- Minimal other content

## Layout Selection Guide

When AI recommends layouts, consider:

| Content Type | Recommended Layouts |
|-------------|-------------------|
| Article cover | M01, S03, M09 |
| List/guide | M05, M08, S06, S09 |
| Comparison | M15, S05, S10 |
| Data/metrics | M10, S01, S07, S02 |
| Timeline/history | M16, S08 |
| Person/team | M14, S11 |
| Quote/thesis | M04, M13, M09 |
| CTA/promotion | S12, M07 |

AI should recommend top-3 and explain why:

```
根据您的内容，我推荐：
1. M01 Magazine Cover — 最适合作为文章封面，大标题+图片
2. M05 Checklist — 适合展示要点列表
3. M09 Atmospheric Thesis — 适合突出核心观点

您想用哪个？
```
